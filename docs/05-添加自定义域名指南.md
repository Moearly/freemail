# 添加自定义域名指南

## 📋 概述

当前 Worker 访问地址：`https://mailfree.gmartnlei.workers.dev`

如果想使用自己的域名（如 `mail.codewith.site`）访问服务，可以通过添加自定义域名实现。

---

## 🎯 方式一：通过 Cloudflare Dashboard（推荐）

### 步骤 1：进入 Worker 设置

1. **访问 Cloudflare Dashboard**
   ```
   https://dash.cloudflare.com
   ```

2. **进入 Workers & Pages**
   - 左侧菜单选择 **"Workers & Pages"**
   - 或直接访问：https://dash.cloudflare.com/[账户ID]/workers-and-pages

3. **选择 Worker**
   - 找到并点击 **`mailfree`** Worker

### 步骤 2：添加自定义域名

1. **进入 Triggers 设置**
   - 点击 **"Settings"** 标签
   - 找到 **"Triggers"** 部分
   - 或点击 **"Triggers"** 标签

2. **添加自定义域名**
   - 在 **"Custom Domains"** 部分
   - 点击 **"Add Custom Domain"** 按钮

3. **输入域名**
   - 域名输入：`mail.codewith.site`（或你想要的子域名）
   - 点击 **"Add Domain"**

### 步骤 3：自动配置 DNS

Cloudflare 会自动完成以下操作：
- ✅ 创建 DNS 记录（CNAME 或 AAAA）
- ✅ 配置 SSL 证书
- ✅ 启用 HTTPS

**等待时间**：通常几秒到几分钟

### 步骤 4：验证配置

1. **检查 DNS 记录**
   - 进入域名的 **DNS** 设置
   - 应该能看到新增的记录：
     ```
     mail.codewith.site  →  CNAME/AAAA  →  指向 Worker
     ```

2. **测试访问**
   - 访问：`https://mail.codewith.site`
   - 应该能正常打开登录页面

## 🎯 方式二：通过 wrangler.toml 配置

### 步骤 1：修改 wrangler.toml

编辑 `wrangler.toml` 文件，添加路由配置：

```toml
name = "mailfree"
main = "src/server.js"
compatibility_date = "2024-10-01"
workers_dev = true

# 自定义域名路由
[[routes]]
pattern = "mail.codewith.site/*"
zone_name = "codewith.site"

# D1 数据库绑定
[[d1_databases]]
binding = "TEMP_MAIL_DB"
database_name = "maill_free_db"
database_id = "9ed54bdc-edf1-4de0-9eed-c1b26825c946"

# 其他配置...
```

### 步骤 2：部署更新

```bash
# 进入项目目录
cd /home/martnlei/codeSpace/freemail

# 部署（确保已配置 CLOUDFLARE_API_TOKEN 环境变量）
npx wrangler deploy
```

### 步骤 3：手动添加 DNS 记录（如果自动创建失败）

1. **进入 DNS 设置**
   - Cloudflare Dashboard → 域名 `codewith.site` → DNS

2. **添加记录**
   - 类型：`AAAA`
   - 名称：`mail`
   - IPv6 地址：`100::` （Cloudflare Workers 的虚拟 IP）
   - Proxy 状态：**橙色云朵**（代理开启）

   或者：
   - 类型：`CNAME`
   - 名称：`mail`
   - 目标：`mailfree.gmartnlei.workers.dev`
   - Proxy 状态：**橙色云朵**（代理开启）

3. **保存**

### 步骤 4：验证

访问 `https://mail.codewith.site` 测试

## 🎯 方式三：通过 wrangler 命令行

```bash

cd /home/martnlei/codeSpace/freemail

npx wrangler deploy --route="mail.codewith.site/*" --name=mailfree
```

## 📝 配置说明

### 域名格式

可以使用以下格式：
- ✅ `mail.codewith.site` - 子域名（推荐）
- ✅ `codewith.site` - 根域名
- ✅ `temp.codewith.site` - 其他子域名
- ✅ 多个域名（多次添加）

### 路由模式

```toml
# 精确匹配
pattern = "mail.codewith.site"

# 通配符匹配（推荐）
pattern = "mail.codewith.site/*"

# 路径匹配
pattern = "codewith.site/mail/*"
```

### SSL/TLS 设置

自定义域名会自动启用 HTTPS：
- ✅ 自动签发 SSL 证书
- ✅ 强制 HTTPS 重定向
- ✅ 支持 HTTP/2 和 HTTP/3

## ⚠️ 注意事项

### 1. 域名必须在 Cloudflare 托管

- 域名的 DNS 必须指向 Cloudflare 的 Nameservers
- 你的 `codewith.site` 已经满足条件 ✅

### 2. SSL 证书生效时间

- 通常几分钟内生效
- 最多可能需要 15 分钟

### 3. DNS 传播时间

- Cloudflare 内部：几秒
- 全球 DNS：几分钟到几小时

### 4. 邮件接收不受影响

- 邮件仍然发送到 `*@codewith.site`
- 自定义域名只影响 Web 访问地址

### 5. 可以配置多个域名

可以同时配置多个自定义域名：
- `mail.codewith.site`
- `temp.codewith.site`
- `freemail.codewith.site`

所有域名都指向同一个服务

## 🔍 故障排查

### 问题 1: 添加域名失败

**错误信息**：
```
Domain not found or not accessible
```

**解决方案**：
1. 确认域名在 Cloudflare 账户中
2. 确认域名 DNS 已托管在 Cloudflare
3. 检查账户权限

### 问题 2: 证书未生效

**现象**：访问显示证书错误

**解决方案**：
1. 等待 10-15 分钟
2. 检查 SSL/TLS 设置：
   - Dashboard → 域名 → SSL/TLS
   - 加密模式应为 **"Full"** 或 **"Full (strict)"**

### 问题 3: DNS 解析不正确

**现象**：域名无法访问

**解决方案**：
1. 检查 DNS 记录是否存在
2. 确认 Proxy 状态为**开启**（橙色云朵）
3. 清除本地 DNS 缓存：
   ```bash
   # Windows
   ipconfig /flushdns
   
   # Linux/Mac
   sudo systemd-resolve --flush-caches
   ```

### 问题 4: 访问返回 404

**现象**：域名可以访问，但返回 404

**解决方案**：
1. 检查路由配置是否正确
2. 确认使用通配符：`mail.codewith.site/*`
3. 重新部署 Worker
