## 二、高危漏洞（已全部修复）

### H-1: 密码哈希不安全 → 已升级为 PBKDF2

**修复前**: 使用不加盐的 SHA-256 哈希，极易被彩虹表攻击

**修复后**: 使用 PBKDF2-HMAC-SHA256 + 16字节随机盐 + 10万次迭代

- 新格式: `pbkdf2:<hex_salt>:<hex_hash>`
- 向后兼容: 旧 SHA-256 格式仍可验证（登录后建议引导用户修改密码）
- 恒定时间比较: 避免时序攻击
- 涉及文件: `src/authentication.js`

### H-2: 管理员密码明文比较 → 已改为哈希验证

**修复前**: `password === ADMIN_PASSWORD` 明文直接比较

**修复后**: 调用 `verifyPassword()` 进行哈希验证，同时兼容明文密码（方便初始配置）

- 涉及文件: `src/routes.js`

### H-3: URL 参数传递管理员令牌 → 已移除

**修复前**: 允许 `?admin_token=xxx` 在 URL 中传递管理员凭据

**修复后**: 仅保留 `Authorization: Bearer` 和 `X-Admin-Token` 头部方式

- 涉及文件: `src/routes.js` — `checkRootAdminOverride()`

### H-4: JWT 签名密钥复用为管理员令牌

**状态**: 部分缓解 — 移除了 URL 参数泄露途径

**建议**: 后续创建独立的 `ADMIN_API_TOKEN` 环境变量，与 `JWT_SECRET` 分离

### H-5: 前端 XSS 漏洞 → 已修复

**修复措施**:
1. 邮件详情弹窗: `sender`、`to_addrs`、`subject` 全部 HTML 转义
2. iframe 渲染: 添加 `sandbox="allow-same-origin"` 阻止脚本执行
3. 涉及文件: `public/js/app.js`、`public/js/landing.js`

## 三、中危漏洞

### M-1: 登录速率限制 → 已实现

**修复**: 基于 `CF-Connecting-IP` 的内存级速率限制

- 窗口: 60 秒
- 限制: 每 IP 最多 10 次登录尝试
- 超限返回: `429 Too Many Requests`
- 涉及文件: `src/routes.js`

### M-2: 邮箱默认密码为邮箱地址本身

**状态**: 未修改 — 这是设计上的兼容性选择

**风险**: 知道邮箱地址即可登录（如果 `can_login=1`）

**缓解**: 管理员可为邮箱设置自定义密码；公开 Landing 模式不需要登录

### M-3: JWT payload 冗余解码 → 已修复

**修复**: 发件权限检查改用 `getJwtPayload()`（优先使用已验证的 `authPayload`）

- 涉及文件: `src/apiHandlers.js`

### M-4: 错误消息泄露内部细节 → 已修复

**修复**: 所有 catch 块改为通用错误消息 + `console.error()` 记录详情

- 修复位置: 发送邮件、更新邮件、创建用户、更新用户

### M-5: 缺少 CORS 配置

**状态**: 暂未修改

**风险**: 中（Cookie 已设置 `SameSite=Strict`，提供主要防护）

**建议**: 后续添加 CORS 中间件，指定 `Access-Control-Allow-Origin` 为自有域名

### M-6: Landing 公开 API 可被滥用

**状态**: 暂未修改（功能特性）

**缓解**: Cloudflare 自带 DDoS 保护 + 可配置 Rate Limiting 规则

**建议**: 在 Cloudflare Dashboard 配置 Rate Limiting 规则保护 `/api/generate` 和 `/api/create`

### M-7: iframe sandbox → 已修复

**修复**: 添加 `sandbox="allow-same-origin"` 阻止邮件中的恶意脚本执行

### M-8: 批量查询 ID 数量限制 → 已修复

**修复**: `ids.slice(0, 50)` 限制最多 50 个

## 四、低危漏洞

### L-1: Cookie Secure 标志 — 可接受

生产环境 Cloudflare 强制 HTTPS，Cookie 始终带 `Secure` 标志。

### L-2: JWT 有效期 24 小时 — 可接受

临时邮箱服务性质，24 小时有效期合理。

### L-3: JWT 缓存大小限制 → 已修复

**修复**: 缓存超过 1000 条时自动清理至 500 条

### L-4: SQL 排序方向验证 — 安全

当前实现使用三元表达式白名单，无风险。

### L-5: Content-Disposition 文件名 → 已修复

**修复**: 移除 `"`、`\`、`\r`、`\n` 等特殊字符

### L-6: /receive 端点认证检查

**状态**: 功能正常（中间件在前置处理认证）
