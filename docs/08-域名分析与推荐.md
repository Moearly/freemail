# 域名分析与推荐

## 一、当前域名分析

### 1.1 现状

| 项目 | 说明 |
|------|------|
| 当前域名 | `mail.codewith.site` |
| 计划新域名 | `freemail.best` |
| 类型 | 二级域名（子域名）→ 独立域名 |
| 主域名 | `codewith.site` |
| 用途 | 临时邮箱 / 一次性邮件服务 / 持久化邮箱 |

### 1.2 问题分析

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **子域名 SEO 权重低** | 高 | 搜索引擎将子域名视为独立站点，无法继承主域名权重 |
| **域名与产品不相关** | 高 | `codewith.site` 暗示"编程"而非"邮件"，用户认知不匹配 |
| **二级域名信任度低** | 中 | 用户对 `.site` 子域名的信任度低于独立 `.com` 域名 |
| **品牌辨识度差** | 中 | `mail.codewith.site` 过长，不利于口碑传播和记忆 |
| **TLD 竞争力弱** | 中 | `.site` 在搜索排名中的权重普遍低于 `.com`、`.io`、`.email` |
| **不利于邮件可信度** | 高 | 作为临时邮箱服务，域名本身影响用户对服务的信任感 |

### 1.3 结论

**当前域名不利于 SEO 推广和品牌建设**。建议注册独立域名。

## 二、域名推荐

### 2.1 推荐策略

优先级排序：
1. **关键词匹配** — 域名包含 `temp`、`mail`、`email`、`free` 等核心词
2. **简短易记** — 不超过 12 个字符（不含 TLD）
3. **品牌化** — 能作为独立品牌识别
4. **TLD 权重** — 优先 `.com` > `.io` > `.email` > `.app`

### 2.2 推荐域名列表

#### 第一梯队（强烈推荐）

| 域名 | 优势 | 预估年费 |
|------|------|---------|
| `freemail.email` | 完美匹配品牌名和产品功能 | ~$15-25 |
| `getfreemail.com` | `.com` 权重高，品牌一致 | ~$10-15 |
| `freemail.app` | Google 管理的 TLD，强制 HTTPS | ~$15-20 |
| `freemail.io` | 科技圈认可度高 | ~$30-50 |

#### 第二梯队（推荐）

| 域名 | 优势 | 预估年费 |
|------|------|---------|
| `tmpbox.com` | 极短，暗示临时邮箱 | ~$10-15 |
| `quicktemp.email` | 关键词丰富 | ~$15-25 |
| `tempbox.io` | 简短，技术感 | ~$30-50 |
| `dropmail.app` | 动作化命名，有辨识度 | ~$15-20 |

#### 第三梯队（预算方案）

| 域名 | 优势 | 预估年费 |
|------|------|---------|
| `freemail.site` | 保持当前 TLD，低成本迁移 | ~$3-5 |
| `mytemp.email` | 便宜的 `.email` 域名 | ~$10-15 |
| `tempinbox.com` | 关键词精准 | ~$10-15 |

### 2.3 域名注册建议

1. **在 Cloudflare Registrar 注册** — 无加价，续费透明
2. **同时注册 `.com` 和业务 TLD** — 如 `freemail.com` + `freemail.email`
3. **配置域名跳转** — 将旧域名 `mail.codewith.site` 301 重定向到新域名

## 三、迁移方案

### 3.1 迁移步骤

```
1. 在 Cloudflare 注册新域名
2. 配置 DNS 记录（与当前 codewith.site 相同）
3. 在 wrangler.toml 中添加新域名路由
4. 配置 Cloudflare Email Routing（新域名）
5. 更新 MAIL_DOMAIN 环境变量（添加新域名）
6. 配置 301 重定向：mail.codewith.site → 新域名
7. 更新 SEO 标签（canonical URL、OG URL 等）
8. 向 Google Search Console 提交新域名
9. 保持旧域名重定向至少 6-12 个月
```

### 3.2 Cloudflare Workers 多域名配置

```toml
# wrangler.toml
routes = [
  { pattern = "mail.codewith.site/*", zone_name = "codewith.site" },
  { pattern = "freemail.email/*", zone_name = "freemail.email" },   # 新域名
]

[vars]
MAIL_DOMAIN = "codewith.site,freemail.email"
```

### 3.3 邮件域名注意事项

新域名需要：
- 配置 MX 记录指向 Cloudflare Email Routing
- 配置 SPF 记录
- 配置 DKIM 记录（如果使用 Resend/Microsoft 发送）
- 配置 DMARC 记录

## 四、总结

| 维度 | 当前 (`mail.codewith.site`) | 推荐 (`freemail.email`) |
|------|---------------------------|----------------------|
| SEO 权重 | 低（子域名） | 高（独立域名） |
| 品牌辨识 | 差 | 优秀 |
| 用户信任 | 低 | 高 |
| 记忆成本 | 高（25字符） | 低（14字符） |
| 年费 | $0（已有） | ~$15-25 |
| 迁移成本 | — | 低（1-2 小时） |

**建议**: 尽快注册 `freemail.email` 或同类域名，作为主域名使用。保持 `mail.codewith.site` 作为 301 重定向源。
