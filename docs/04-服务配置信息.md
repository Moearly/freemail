# FreeMail 服务配置信息

## 基本配置信息

### 1. Worker 域名

**用于 API 请求的 Cloudflare Worker 服务域名**

```
https://mail.codewith.site
```

**备用域名**:
```
https://mailfree.gmartnlei.workers.dev
```

---

### 2. 邮箱域名

**用于生成临时邮箱的域名后缀**

```
codewith.site
```

说明：生成的邮箱格式为 `xxx@codewith.site`，支持多域名配置（`wrangler.toml` 中 `MAIL_DOMAIN` 使用逗号分隔）。

---

### 3. 管理员登录

```
用户名: admin（对应 wrangler.toml 中的 ADMIN_NAME）
密码:   通过 wrangler secret put ADMIN_PASSWORD 设置
```

---

### 4. JWT Token（Root Admin Token）

JWT Token 是超级管理员令牌，用于跳过登录流程直接调用所有 API。

通过 `wrangler secret put JWT_TOKEN` 设置，设置后无法读取原值。

## 认证方式说明

### Web 界面登录

```
访问地址: https://mail.codewith.site
用户名: admin
密码: <你设置的 ADMIN_PASSWORD>
```

登录后浏览器通过 Cookie（`iding-session`）维持会话。

### API 认证方式

所有需要认证的 API 接口，通过携带 JWT Token 实现鉴权：

#### 方式 1：Authorization Bearer Token（推荐）
```bash
curl -H "Authorization: Bearer <YOUR_JWT_TOKEN>" \
  "https://mail.codewith.site/api/generate"
```

#### 方式 2：X-Admin-Token Header
```bash
curl -H "X-Admin-Token: <YOUR_JWT_TOKEN>" \
  "https://mail.codewith.site/api/generate"
```

#### 方式 3：URL Query Parameter
```bash
curl "https://mail.codewith.site/api/generate?admin_token=<YOUR_JWT_TOKEN>"
```

> 注意：以上三种方式等效，携带的值必须与 Worker Secrets 中的 `JWT_TOKEN` 完全一致。

## 环境变量与密钥总览

### wrangler.toml 公开变量

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `MAIL_DOMAIN` | `codewith.site` | 邮箱域名，多域名用逗号分隔 |
| `ADMIN_NAME` | `admin` | 管理员用户名 |
| `LANDING_PUBLIC` | `true` | Landing Page 匿名访问开关 |
| `MAIL_EXPIRE_HOURS` | `24` | 邮件自动过期时间（小时） |

### Secrets（通过 wrangler secret put 设置）

| 密钥名 | 必填 | 说明 |
|--------|------|------|
| `ADMIN_PASSWORD` | 是 | 管理员登录密码 |
| `JWT_TOKEN` | 是 | JWT 签名密钥 / Root Admin Token |
| `GUEST_PASSWORD` | 否 | 访客模式密码 |
| `RESEND_API_KEY` | 否 | Resend 发件密钥，支持多域名格式: `domain1=key1,domain2=key2` |
| `MS_ACCOUNTS` | 否 | 微软邮箱发件配置（JSON 数组）|
| `FORWARD_RULES` | 否 | 邮件转发规则 |

## 配置修改指南

### 修改管理员密码

```bash
cd /home/martnlei/codeSpace/freemail
echo "你的新密码" | npx wrangler secret put ADMIN_PASSWORD
```

### 重新生成 JWT Token

```bash
# 生成新 Token
NEW_TOKEN=$(openssl rand -hex 32)
echo "新的 Token: $NEW_TOKEN"

# 更新到 Worker
echo "$NEW_TOKEN" | npx wrangler secret put JWT_TOKEN
```

> 重要：JWT Token 是加密存储的，设置后无法读取原值，请自行备份。

### 配置邮件转发

```bash
# KV 格式：prefix=target_email
echo "xms=your@gmail.com,*=catch-all@gmail.com" | npx wrangler secret put FORWARD_RULES

# 或 JSON 格式
echo '[{"prefix":"xms","email":"your@gmail.com"},{"prefix":"*","email":"catch-all@gmail.com"}]' | npx wrangler secret put FORWARD_RULES
```

### 配置微软邮箱发件

```bash
echo '[{"email":"user@outlook.com","tenantId":"xxx","clientId":"xxx","clientSecret":"xxx"}]' | npx wrangler secret put MS_ACCOUNTS
```

| 类型 | 绑定名 | 资源 | 说明 |
|------|--------|------|------|

| 静态资源 | `ASSETS` | `public/` 目录 | 前端 HTML/CSS/JS |

## 安全提示

1. 不要将密码和 Token 提交到公开仓库
2. 定期更换密码和 Token
3. 所有密钥通过 `wrangler secret put` 设置，不要写在 `wrangler.toml` 中
4. `CLOUDFLARE_API_TOKEN` 只在本地部署时使用，不要写入文档或代码
