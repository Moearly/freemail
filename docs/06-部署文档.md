## 📝 部署步骤记录

### 1. 环境准备

#### 1.1 安装 Node.js
```bash
# 使用 nvm 安装 Node.js v20
nvm install 20
nvm use 20
```

**说明**: Wrangler CLI 要求 Node.js 版本 >= v20.0.0

#### 1.2 安装 Wrangler CLI
```bash
npm install -g wrangler
```

### 2. Cloudflare 账户认证

#### 2.1 创建 API Token

由于 WSL 环境中 OAuth 回调无法正常工作，采用 API Token 认证方式。

**Token 创建步骤**：
1. 访问 https://dash.cloudflare.com/profile/api-tokens
2. 点击 "Create Token"
3. 选择 "Edit Cloudflare Workers" 模板
4. 额外添加权限：
   - User → User Details: Read
   - User → Memberships: Read
   - Account → D1: Edit
   - Account → Workers R2 Storage: Edit
5. 创建并复制 Token

#### 2.2 配置 Token
```bash
export CLOUDFLARE_API_TOKEN="your_token_here"
```

**最终使用的 Token**: `mvJNgJxA16kKLvW0t2sJ5gNuoCorqKy0DS-IGGxB`

### 3. 创建 D1 数据库

#### 3.1 创建数据库
```bash
wrangler d1 create maill_free_db
```

**输出**：
```
Database created successfully
Database ID: 9ed54bdc-edf1-4de0-9eed-c1b26825c946
Region: APAC
```

#### 3.2 更新 wrangler.toml
```toml
[[d1_databases]]
binding = "TEMP_MAIL_DB"
database_name = "maill_free_db"
database_id = "9ed54bdc-edf1-4de0-9eed-c1b26825c946"
```

#### 3.3 初始化数据库表结构
```bash
# 本地初始化
wrangler d1 execute maill_free_db --file=./d1-init-basic.sql

# 远程初始化
wrangler d1 execute maill_free_db --remote --file=./d1-init-basic.sql
```

**结果**: 16 条 SQL 语句执行成功，创建了以下表：
- `mailboxes` - 邮箱表
- `messages` - 邮件表
- `sent_emails` - 发件记录表
- `users` - 用户表
- `user_mailboxes` - 用户邮箱关联表

### 4. 创建 R2 存储桶

#### 4.1 启用 R2 服务
在 Cloudflare Dashboard 中启用 R2 服务（需要绑定信用卡，但有免费额度）

#### 4.2 创建存储桶
```bash
wrangler r2 bucket create mail-eml
```

**R2 免费额度**（每月）：
- 存储: 10 GB
- Class A 操作（写入）: 100 万次
- Class B 操作（读取）: 1000 万次

### 5. 配置环境变量

#### 5.1 配置必需的 Secrets
```bash
# 邮箱域名
echo "codewith.site" | wrangler secret put MAIL_DOMAIN

# 管理员密码
echo "Admin@CloudMail2024" | wrangler secret put ADMIN_PASSWORD

# JWT 签名密钥（自动生成）
openssl rand -hex 32 | wrangler secret put JWT_TOKEN
```

#### 5.2 环境变量说明

| 变量名 | 说明 | 是否必需 |
|--------|------|---------|
| `MAIL_DOMAIN` | 邮箱域名，支持多个（逗号分隔） | 是 |
| `ADMIN_PASSWORD` | 管理员密码 | 是 |
| `JWT_TOKEN` | JWT 签名密钥 | 是 |
| `ADMIN_NAME` | 管理员用户名（默认: admin） | 否 |
| `RESEND_API_KEY` | Resend 发件 API 密钥 | 否 |
| `FORWARD_RULES` | 邮件转发规则 | 否 |

### 6. 部署 Worker

#### 6.1 部署命令
```bash
wrangler deploy
```

#### 6.2 部署输出
```
✨ Success! Uploaded 28 files
Total Upload: 139.55 KiB / gzip: 28.72 KiB
Deployed mailfree triggers
  https://mailfree.gmartnlei.workers.dev
```

### 7. 配置邮件路由

#### 7.1 访问邮件路由设置
1. 访问 Cloudflare Dashboard
2. 选择域名 `codewith.site`
3. 进入 Email → Email Routing

#### 7.2 配置 Catch-all 规则
1. 点击 "路由规则" 标签
2. 找到 "Catch-all 地址" 部分
3. 点击 "编辑"
4. 配置如下：
   - 操作: **发送到 Worker**
   - Worker: **mailfree**
   - 状态: **启用**（绿色开关）
5. 保存配置

#### 7.3 验证配置
确认显示：
```
Catch-All | 操作: 发送到 Worker | 目标: mailfree | 状态: 活动
```

## ⚠️ 部署过程中遇到的问题

### 问题 1: OAuth 登录失败

**现象**: 
```
ERR_CONNECTION_CLOSED
无法访问 welcome.developers.workers.dev
```

**原因**: WSL2 环境中浏览器无法访问 WSL 内部的 `localhost:8976` 回调地址

**解决方案**: 改用 API Token 认证方式
```bash
export CLOUDFLARE_API_TOKEN="your_token"
```

---

### 问题 2: API Token 权限不足

**现象**: 
```
Authentication error [code: 10000]
A request to the Cloudflare API (/accounts/.../d1/database) failed
```

**原因**: 
- 初始 Token 缺少必要权限
- 使用 "Cloudflare Workers" 模板创建的 Token 默认权限不足

**解决方案**: 创建 Token 时额外添加权限
- User → User Details: Read ✅
- User → Memberships: Read ✅
- Account → D1: Edit ✅
- Account → Workers R2 Storage: Edit ✅

**最终正确的 Token**: `mvJNgJxA16kKLvW0t2sJ5gNuoCorqKy0DS-IGGxB`

---

### 问题 3: R2 服务未启用

**现象**: 
```
Please enable R2 through the Cloudflare Dashboard. [code: 10042]
```

**原因**: R2 是可选服务，需要在 Dashboard 中手动启用

**解决方案**: 
1. 访问 Cloudflare Dashboard
2. 进入 R2 页面
3. 点击 "Enable R2"
4. 绑定信用卡（有免费额度）
5. 创建存储桶

---

### 问题 4: WSL SSL 连接失败

**现象**: 
```
curl: (35) error:0A000126:SSL routines::unexpected eof while reading
net::ERR_CONNECTION_CLOSED
```

**原因**: WSL2 的网络栈在某些 SSL 握手场景下存在兼容性问题

**影响**: 
- 无法从 WSL 环境访问已部署的 Worker
- 不影响实际部署和服务运行

**验证方案**: 在 Windows 宿主机浏览器中访问，服务正常运行

**技术说明**: 
- Worker 服务本身运行正常
- 问题仅限于 WSL → Cloudflare Workers 的 SSL 连接
- 外部用户访问不受影响

---

### 问题 5: Node.js 版本不兼容

**现象**: 
```
Wrangler requires at least Node.js v20.0.0. You are using v19.9.0
```

**原因**: Wrangler 4.x 要求 Node.js >= v20

**解决方案**: 
```bash
nvm install 20
nvm use 20
```

**注意**: 切换 Node 版本后需要重新安装 wrangler
```bash
npm install -g wrangler
```

## 💡 常见问题

### Q1: 为什么不需要域名解析？

**答**: Worker 使用的是 Cloudflare 提供的 `*.workers.dev` 子域名，这是 Cloudflare 自动管理的域名，无需手动配置 DNS 解析。

**说明**:
- `mailfree.gmartnlei.workers.dev` 是自动分配的
- 如果要使用自定义域名（如 `mail.codewith.site`），才需要在 Worker 设置中添加自定义域名（Cloudflare 会自动创建 DNS 记录）

### 如何添加自定义域名？

请参考下一节：**添加自定义域名指南**

### Q2: 收不到邮件怎么办？

**检查清单**:
1. 确认 Email Routing 已启用
2. 确认 MX 记录已配置（Cloudflare 自动配置）
3. 确认 Catch-all 规则已设置为发送到 `mailfree` Worker
4. 等待 DNS 生效（通常几分钟到几小时）

### Q3: 如何查看 Worker 日志？

```bash
export CLOUDFLARE_API_TOKEN="your_token"
wrangler tail --format pretty
```

### Q4: 如何更新 Worker？

修改代码后重新部署：
```bash
wrangler deploy
```

### Q5: 如何删除 Worker？

```bash
wrangler delete mailfree
```
